# Heart Landing — Vue-версия

Это Vue 3 + Vite перенос вашего оригинального проекта
`danilamisocenko-hue/heart` (React/TSX, "Heart Landing").

## Что перенесено 1:1

- Экран-терминал: печатающийся текст `Initializing heart.PROTOCOL_v2.0...`,
  статус `READY`, кнопка `Decrypt Message` с иконкой замка, подсказка
  `(or just click anywhere)`.
- По клику (на кнопку или в любое место экрана, когда терминал
  закончил печатать) — переход к сердцу.
- Сердце на `<canvas>`, «сплетённое» из множества повторений текста,
  летящих по параметрической кривой сердца, с плавным появлением
  каждой точки в случайном порядке — код перенесён из
  `TextHeart.tsx` практически построчно.
- Подпись `Decrypted`, тонкая линия-разделитель, кнопка `Re-encrypt`
  для возврата назад.
- Технические подписи по углам (`ln: 420`, `id: 0xDEADBEEF`,
  `heart_reveal // success`), эффект скан-линии, свечение текста,
  палитра (`#050505` фон, `#ff4d6d` розовый), шрифты Fira Code / Inter.
- Вёрстка на Tailwind CSS v4 — те же классы, что в оригинале (замена
  `className` → `class`).

Framer Motion (`motion/react`) в оригинале заменён на встроенные
Vue-переходы (`<Transition>`) — визуально те же fade / fade-up /
scale-эффекты, без лишней зависимости.

## Что изменено по вашей просьбе

- Текст внутри сердца: `i love you` → **`Я люблю тебя`**
  (меняется в `src/components/TextHeart.vue`, константа `HEART_TEXT`).

## Что добавлено сверх оригинала

В самом репозитории музыки и 3D-вращения не было. По вашей просьбе
добавлено:

**Музыка.** Играет с момента раскрытия сердца (клик по "Decrypt
Message" / по экрану). Кладёте свой легально полученный файл в
`public/angel.mp3`. Константы `CHORUS_START` / `CHORUS_END` в начале
`<script setup>` файла `src/App.vue` задают тайм-код припева для
**полной** песни (сейчас ориентировочно ~4:18–5:40 — обычный формат
Angel идёт 6:17, подстройте под свой файл). **Если же ваш файл — уже
готовый короткий отрывок** (например, вы сами вырезали 10 секунд с
"Love you, love you..."), это автоматически определяется по реальной
длительности файла, и играть будет весь этот отрывок целиком, без
попытки перемотать за его пределы.

**Кнопка выключения звука** в углу экрана + мета-теги под iPhone
(safe-area для чёлки/индикатора дома).

**3D-сердце вместо плоского.** В оригинальном репозитории, помимо
плоской canvas-версии (`TextHeart.tsx`, которая реально используется
на сайте), нашёлся ещё неподключённый компонент `HeartScene.tsx` —
3D-облако из текста на Three.js с вращением и звёздами. Он перенесён
на Vue как `src/components/TextHeart3D.vue` и **теперь используется
вместо плоской версии** — по нему можно провести пальцем, чтобы
покрутить сердце в 3D, оно само медленно вращается в покое, и
добавлен лёгкий параллакс: если наклонять телефон, камера немного
"заглядывает" за сердце. На iOS при первом раскрытии система один раз
спросит разрешение на доступ к датчикам движения — это нужно для
параллакса, без него сайт тоже прекрасно работает, просто без наклона.
Старый плоский вариант (`TextHeart.vue`) остался в проекте на случай,
если 3D покажется слишком тяжёлым для какого-то устройства — просто
верните `import TextHeart from './components/TextHeart.vue'` и
`<TextHeart />` в `App.vue`.

Если музыка не нужна вовсе — просто не кладите файл в `public/`, всё
остальное будет работать как обычно.

## Исправления по итогам тестирования на реальном сайте

- **Зацикливание музыки.** Короткий отрывок теперь зацикливается плавно:
  на стыке цикла громкость на 0.26 сек гаснет и снова нарастает (без
  щелчка на стыке), а с каждым повтором звук становится немного громче
  — константы `BASE_VOLUME` / `VOLUME_STEP` / `MAX_VOLUME` / `FADE_MS`
  в начале `<script setup>` файла `src/App.vue`.
- **Кнопка "Re-encrypt" на телефоне.** Область нажатия была только по
  границам текста 10px — на телефоне пальцем почти невозможно попасть.
  Увеличена невидимая область нажатия (минимум 44×44px), сам текст
  визуально не изменился.
- **Сердце читалось как каша из букв.** В 3D-версии надписи были в
  5-6 раз крупнее, чем нужно относительно размера сердца, и полностью
  перекрывали друг друга. Уменьшены до пропорции как в оригинальном
  `HeartScene.tsx` — теперь сердце должно читаться как форма.

## Ещё раунд правок

- **Музыка наконец зацикливается железно.** Была гонка: проверка "пора
  зацикливать" стояла всего за 0.26 сек до конца файла, и на коротких
  клипах браузер иногда не успевал её поймать между событиями
  `timeupdate` — аудио доигрывало до конца и просто останавливалось.
  Увеличен запас + добавлена страховка на событие `ended` (если всё же
  проскочили момент — зацикливаем немедленно).
- **Сердце ближе и надпись крупнее.** Камера была на дистанции 5 —
  придвинул до 3.2. Размер текста в прошлый раз ушёл в другую крайность
  (стало слишком мелко, сердце казалось пустым) — вернул к пропорции
  ближе к оригиналу, плюс увеличил число частиц для плотности.
- **Убраны "Decrypted", "Re-encrypt" и технические подписи по углам.**
  После раскрытия теперь просто сердце, музыка и кнопка звука — без
  кнопки возврата на терминал (перезагрузка страницы = снова терминал).

## Установка и запуск

Нужен Node.js 18+.

```bash
npm install
npm run dev
```

Откроется `http://localhost:5173`.

### Открыть на iPhone во время разработки

```bash
npm run dev -- --host
```

и открыть показанный адрес `http://192.168.x.x:5173` в Safari на
телефоне (в одной Wi-Fi сети).

### Деплой на GitHub Pages — в ваш репозиторий

Проект уже настроен для `https://github.com/BottomSheet/love.git` —
сайт будет жить на **https://bottomsheet.github.io/love/**
(`vite.config.js` уже знает про этот путь, ничего менять не нужно).

**Шаг 1.** Положите `public/angel.mp3` (он не входит в присланный
архив — добавьте сами перед коммитом).

**Шаг 2.** Отправьте код в репозиторий:

```bash
cd heart-clone
git init
git remote add origin https://github.com/BottomSheet/love.git
git add .
git commit -m "Heart landing: терминал, 3D-сердце, музыка"
git branch -M main
git push -u origin main
```

Если в репозитории `love` уже что-то есть (например, автосозданный
GitHub-ом README) и push выдаст ошибку про расхождение веток —
сначала:

```bash
git pull origin main --allow-unrelated-histories
```

решите конфликт (скорее всего только в `README.md`), затем `git push`
ещё раз. Если там точно ничего ценного нет — можно и через
`git push -u origin main --force`.

**Шаг 3.** Включите Pages (один раз, на сайте GitHub):
откройте `https://github.com/BottomSheet/love/settings/pages` → в
блоке **Build and deployment → Source** выберите **GitHub Actions**
(не "Deploy from a branch") → сохраните.

Дальше при каждом `git push` в `main` сайт сам пересобирается и
обновляется (см. `.github/workflows/deploy.yml`) — через 1-2 минуты
после пуша откройте **https://bottomsheet.github.io/love/**.

### Если музыка всё ещё не играет — проверьте по порядку

1. **Переключатель звонок/беззвучный на iPhone.** Safari на iOS по
   умолчанию заглушает `<audio>`, если телефон в беззвучном режиме —
   самая частая причина. Переведите в режим звонка, прибавьте громкость.
2. **Файл лежит именно в `public/angel.mp3`.** После `npm run build`
   он должен появиться в `dist/angel.mp3`.
3. **`npm install` заново**, если тестировали более старую версию —
   в этой добавилась библиотека `three` для 3D-сердца.
4. **Консоль браузера.** На компьютере — F12 → Console. На iPhone —
   Настройки → Safari → Дополнения → включить "Веб-инспектор",
   подключить к Mac по кабелю, открыть вкладку через меню Develop в
   Safari на Mac. Если файл не грузится/не декодируется — там будет
   точное сообщение (добавлено специально для этого).
